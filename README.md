### 5. Причины успешного выполнения тестов

1. **Debit_WithValidAmount_UpdatesBalance** —  успешно  
    Исправили ошибку в коде

2. **Debit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange** —  успешно  
    В методе стоит проверка `if (amount < 0)`, которая выбрасывает исключение. Тест это проверяет и получает ожидаемую ошибку

3. **Debit_WhenAmountIsMoreThanBalance_ShouldThrowArgumentOutOfRange** —  успешно  
    В методе стоит проверка `if (amount > m_balance)`, которая не дает снять больше, чем есть на счете

4. **Credit_WithValidAmount_UpdatesBalance** —  успешно  
    Метод правильно прибавляет деньги к балансу

5. **Credit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange** —успешно  
    В методе есть защита от отрицательных сумм. Тест проверяет, что программа ругается при попытке пополнить счет "минусовыми" деньгами

6. **Credit_WithZeroAmount_DoesNotChangeBalance** —  успешно  
   Пополнение на 0 рублей не меняет баланс
